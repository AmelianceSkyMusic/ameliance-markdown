### Правила роботи з картками todo

- Кожна картка має:
  1. **Name (title)** — коротка, конкретна назва задачі (що зробити). Максимум 5-7 слів.
  2. **Body (page content)** — докладний prompt для агента у вигляді параграфа (block type: paragraph) на сторінці картки. Prompt має бути самодостатнім — опис задачі, вимоги, контекст, технічні деталі, щоб агент міг виконати задачу без додаткових уточнень.
  3. **Type (multi_select)** — кожна картка повинна мати тип: `bug` або `feat`. Якщо картка не відповідає жодному з цих типів — залишай Type порожнім.
  При створенні нової картки заповнюй обидва поля: спочатку створи сторінку з Name, потім додай prompt через `notion_API-patch-block-children` (або `API-patch-block-children` залежно від MCP-клієнта). Prompt бери з оригінального опису користувача, адаптуючи його в чітку інструкцію. Тип встановлюй одразу після створення.
- При створенні нової картки або зміні статусу — **переміщуй картку на самий верх** бази даних.
В API: онови `Last edited time` через `notion_API-patch-page` (або `API-patch-page` залежно від MCP-клієнта) з будь-яким проперті (наприклад,
`{"Type":{"multi_select":[]}}`), щоб вона піднялася вгору при сортуванні за `last_edited_time`.
- **Ніколи не виконуй `git commit`, `git push` або `git add` без явного дозволу користувача.** Ти можеш показувати diff і пропонувати, але не застосовуй сам.

Перед виконанням будь-яких змін спочатку перевіряй Notion сторінку ameliance-markdown на актуальні інструкції, специфікації, задачі та вимоги.
